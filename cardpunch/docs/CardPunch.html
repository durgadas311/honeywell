Ctrl-AToggle between Program Card and current data card. The column indicator is highlighted yellow when the program card is in the punch station.

Note, there is no "Feed" key. Pressing "Release" will feed a new card. "Auto Feed" affects handling at the end of a card after manually punching, or "Skip" or "Dup".

Menus

Select a file to receive punched cards

Set output hopper to disard all punched cards
File->Input
Select file to use as input (pre-punched cards)
File->Blank (default)
Set input to an (endless) stack of blank cards

Load program card from previously saved file
Prog->Save
Save current program card into file

Internals

Punched cards are saved as 80 16-bit integers, with bits 11-0 having the punched rows (bit 11 = 12-Punch, bit 10 = 11-Punch, bit 9 = Zero-Punch, ... bit 0 = Nine-Punch). Bit 12 indicates whether the column should be printed (printer was on when the column was punched, or the column was later run through the interpreter).

Note that printing does not overstrike. When a column is multi-punched, the printed symbol changes as punches are added. Invalid (unknown) punch combinations are shown as blank.
