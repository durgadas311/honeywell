Punch Card Accounting Machine Help

This is currently a minimal implementation of the IBM 402/403 Accounting machines.

The input hopper is located in the upper left. Output stacker is located derectly below the hopper. The program panel is implemented as a properties file. currently, this file must be editted external from the simulation. The START, STOP, and FINAL TOTAL buttons should function as on original equipment, with the exception that FINAL TOTAL does not requires simultaneous pressing of START. The run rate is set to 20 cards per second.

The hopper/stacker are set to show 100 cards, with each card represented by a line of pixels. More cards may be in the hopper/stackers, in fact there is no practical limit (unlike original equipment). When a hopper/stacker has exceeded its display limit, there will be a red line on top of the stack.

Clicking on the input hopper will start a dialog to add a PCD file to the hopper. By default, this will add the new file after all existing files in the hopper, but the option "Remove All" may be used to clear the hopper first.

Clicking on the stacker will empty it. This is for appearances sake only.

Programming is done by a "properties" file, that contains lines of the format "name = value". Referencing the original equipment programming panel, the following are supported:

PRINT, COUNTER, and COMPARING hubs (as well as LIST) relate to property names, others appear only in property values. COUNTER hubs can be used in both contexts.

COUNTER CONTROL and CARRY are specified in the counter propery, and the "carry=" parameter designates which counter will receive the carry from this counter.

In the property value, a set of hubs (a field) may be designated by adding an asterisk and the number of hubs in the field. It is generally expected that all sets of hubs in a given property will be the same width, and that width determines the property width.

Also note that "list=all" or "list=cards" is required in order to print on every card.

Example 1: comparing a field (current and previous card). Field starts at card column 5 and is 4 columns wide. Compare triggers "minor" group. 

    c1 = 2.5*4 3.5*4 start=minor

    Which means to send card columns 5-8 from SECOND READING to COMPARING hubs 1-4, and send card columns 5-8 from THIRD READING to the other COMPARING hubs 1-4. COMPARING EXIT hubs 1-4 are linked together, and the output is tied to the MINOR START hub.

Example 2: Add field from card column 5, 4 columns wide, to counter 8B and print result into numeric columns 10-17 with zero suppression. Print/reset on minor group change. Note how the 4-col field is positioned to the lower part of the 8-digit counter. 

    8b5 = 3.5*4 plus=all total=minor
n10 = 8b1*8 zero

    Line 1 sends card columns 5-8 from THIRD READING to COUNTER 8B ENTRY hubs 5-8, wires ALL CYCLES to the PLUS hub for COUNTER 8B, and wires the TOTAL hub for 8B to the MINOR TOTAL PROGRAM hub.

    Line 2 wires COUNTER 8B EXIT hubs 1-8 to NUMERICAL PRINT columns 10-17 and enables zero suppression.

    Note, zero suppression was configured by changing levers on the type bars in the printer (and not by wires on the control panel). This property parameter defines zero suppression for the field automatically, which is a compromise.
